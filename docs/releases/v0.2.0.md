# Release Notes: v0.2.0

**Release Date:** 2026-06-07

## Overview

This release adds powerful developer experience features: operation scaffolding, schema validation, drift detection, and file watching for automatic regeneration. The generate command now uses genqlient as an embedded library for better performance and error handling.

## Highlights

- **Scaffold Operations** - Generate stub queries and mutations from schema types
- **Validate Operations** - Check operations against schema before generation
- **Schema Drift Detection** - Compare local schema to remote endpoint
- **Watch Mode** - Auto-regenerate on file changes
- **Embedded genqlient** - No longer requires genqlient CLI installation

## New Commands

### `scaffold` - Generate Stub Operations

Generate template operations from Query and Mutation types:

```bash
# Generate all operations
gqlforge scaffold schema.graphql -o operations/

# Only queries with depth 3
gqlforge scaffold schema.graphql --type Query --depth 3

# Filter by field name patterns
gqlforge scaffold schema.graphql --include "user*,feature*"
gqlforge scaffold schema.graphql --exclude "internal*,deprecated*"
```

**Flags:**

| Flag | Description | Default |
|------|-------------|---------|
| `--type` | Type to scaffold: Query, Mutation, or both | both |
| `--depth` | Max field depth for selection sets | 2 |
| `--include` | Glob patterns for fields to include | |
| `--exclude` | Glob patterns for fields to exclude | |
| `-o, --output` | Output directory | operations |

### `validate` - Validate Operations

Check operations against schema before generation:

```bash
# Validate with defaults
gqlforge validate

# Custom paths
gqlforge validate --schema schema.graphql --operations "queries/*.graphql"

# Strict mode (warnings become errors)
gqlforge validate --strict

# JSON output for CI/CD
gqlforge validate --json
```

**Flags:**

| Flag | Description | Default |
|------|-------------|---------|
| `--schema` | Path to schema file | schema.graphql |
| `--operations` | Glob pattern for operations | operations/*.graphql |
| `--strict` | Treat warnings as errors | false |
| `--json` | Output as JSON | false |

### `diff` - Schema Drift Detection

Compare local schema to remote endpoint:

```bash
# Basic diff
gqlforge diff schema.graphql https://api.example.com/graphql --token xxx

# Only show breaking changes
gqlforge diff schema.graphql https://api.example.com/graphql --breaking-only

# JSON output for CI/CD
gqlforge diff schema.graphql https://api.example.com/graphql --json
```

**Output:**

```
Schema Diff: schema.graphql vs https://api.example.com/graphql

+ Added Types:
  + AiAgent
  + AiAgentPage

- Removed Types (BREAKING):
  - LegacyReport

~ Changed Types:
  ~ Feature:
    + newField
    - deprecatedField (BREAKING)
    ~ status: Status → StatusEnum (BREAKING)

Summary: 2 added, 1 removed, 3 breaking changes
```

### `watch` - Auto-Regeneration

Watch files and regenerate on changes:

```bash
# Watch with defaults
gqlforge watch

# Custom config
gqlforge watch --config genqlient.yaml

# Custom debounce
gqlforge watch --debounce 500ms
```

**Flags:**

| Flag | Description | Default |
|------|-------------|---------|
| `--config` | Path to genqlient.yaml | genqlient.yaml |
| `--debounce` | Debounce duration | 300ms |

## Breaking Changes

### Embedded genqlient

The `generate` command now uses genqlient as a library instead of calling it via `exec.Command`. This means:

- **No longer requires** `genqlient` CLI to be installed
- Better error messages with full context
- Faster generation (no process spawning)
- Version locked to go.mod

If you relied on a specific genqlient CLI version, note that gqlforge now uses genqlient v0.8.1.

## Updated Workflow

```
GraphQL Endpoint
      ↓
gqlforge introspect      ← Fetch schema
      ↓
schema.graphql
      ↓
gqlforge scaffold        ← NEW: Generate stub operations
      ↓
operations/*.graphql     ← Edit generated stubs
      ↓
gqlforge validate        ← NEW: Check before generation
      ↓
gqlforge generate        ← Now uses embedded genqlient
      ↓
generated/client.go

# For ongoing development:
gqlforge watch           ← NEW: Auto-regenerate on changes
gqlforge diff            ← NEW: Detect schema drift
```
